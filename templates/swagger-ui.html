Description and tests of HTTP requests

GET Categories:

  • GET /api/categories/ || Returns all product categories

    • Parametros:


  • id: Category id
  • name: Product category name


    • GET Unique category:

  • GET /api/categories/6/ || Returns a specific category

    • Parametros:


  • id: Category id
  • name: Product category name


    • GET Products:

  • GET /api/search/ || Return all products

    • Parametros:


  • id: Product id
  • name: Product name
  • url_image: Product image url
  • price: Product price
  • discount: Product discount in whole numbers (0-100)
  • category: Category id to which the product belongs

    • Optional parameters:
  • GET /api/search/?ordering=price&search=pisco
    • "ordering" ordering method for the products according to the following options:
      • "price": price from lowest to highest
      • "-price": price from highest to lowest
      • "name": A-Z name
      • "-name": Z-A name
      • "discount": discount from lowest to highest
      • "-discount": discount from highest to lowest
    • "search" keyword to match product names


    • GET Unique product:

  • GET /api/search/7/ || Return a specific product

    • Parametros:


  • id: Product id
  • name: Product name
  • url_image: Product image url
  • price: Product price
  • discount: Product discount in whole numbers (0-100)
  • category: Category id to which the product belongs


    • In the API section of Bsale_test, you can make requests where the structures that return the requests for each URL are shown in detail; In addition to being able to see examples, do tests and see the responses of the server.
